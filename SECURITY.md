# 安全政策

## 报告安全问题

请通过 GitHub 仓库的 **Security → Report a vulnerability** 私下提交安全问题。不要在公开 Issue 中披露 API Key、Token、可利用细节或个人交易数据。

报告中请包含受影响版本、复现条件、潜在影响和建议修复方式。维护者确认后会尽快给出处理进度。

## 凭据与本地数据

- API Key 只能放在本地 `.env` 或安全的部署环境变量中。
- `.env`、行情快照、新闻缓存和本地数据库默认不会提交到 Git。
- 前端不得读取或代理输出模型与数据源密钥。
- 如果密钥曾出现在终端输出、截图、提交历史或公开聊天中，应立即在服务商后台撤销并重新生成。

本政策不覆盖行情准确性、投资收益或第三方免费接口的可用性；这些边界详见 README 风险声明。
