# 数据接入清单

下面列出生产接入所需的能力，而不是把供应商写死进业务代码。所有适配器都必须在 `providers/` 中归一化为领域模型。

| 能力 | 何时使用 | 最低接口要求 | MVP 选择 |
|---|---|---|---|
| 全市场收盘快照 | 每交易日 15:01 后缓存 | 一次返回 A 股代码、名称、完整开高低收、成交量、成交额 | `sina_free` 为当前默认；`eastmoney_free` 保留给未来实时链路；AKShare 为兼容适配器 |
| 历史日 K | 收盘后缓存；候选股研究时读取 | 至少 250 个交易日 OHLCV、复权方式、交易日 | 已实现 `eastmoney_free` / `sina_free` 候选股日 K；AKShare/Tushare Pro 可作备选 |
| 同时段量能 | 候选研究 | 历史 1 分钟或 5 分钟累计量/额 | Tushare Pro 分钟权限或已授权行情源 |
| 新闻与公告 | 盘后、盘前、14:30 | 标题、正文或摘要、发布时间、URL、公司/主题映射 | 已实现：CNINFO 候选公司公告（官方 PDF 链接）+ AKShare/东方财富候选股新闻；可选 Tushare `major_news` 精确代码/全称归因 |
| 板块/概念 | 候选研究 | 行业、概念成分、板块涨幅、板块资金 | AKShare 原型；授权源更稳定 |
| 资金流 | 候选研究 | 个股主力净流、北向、融资余额；时间戳 | Tushare Pro / 授权数据源 |
| 基本面 | 每日或每周缓存 | PE、PB、ROE、营收/利润增速、披露期 | Tushare Pro / 财报数据源 |
| LLM 研究 | 仅研究池 | 受 JSON Schema 约束的模型 API；不得使用自由文本作为决策输入 | OpenAI / 其他兼容模型 |

## 免费公开源的运行策略

`eastmoney_free` 是当前第一优先级的免费 Provider，不依赖 AKShare 或其他第三方 Python 包。为降低连续全市场分页导致的连接重置，它固定为单线程、每页 0.4 秒间隔、最多 3 次指数退避重试；约 56 页的完整快照通常需要约 25 秒以上。它不做并发抓取，也不会在缺页后悄悄返回已有数据。

`sina_free` 是当前默认的前收盘筛选 Provider。它在收盘后缓存完整快照；下个交易日的研究流程自动读取最近缓存，并只在形态筛选后拉取候选股历史 K。它同样顺序分页、重试、检查最少股票数和代码唯一性。该来源的历史 K 没有历史成交额，因此历史成交额字段明确记为 0，涉及历史成交额的分析必须等待更完整的数据源。

免费公开端点没有可依赖的服务等级保证，也不等同于可商用的数据授权。系统把它定位为个人、低频、人工复核的研究输入；运行前仍应核对数据源的最新使用条款。若连续失败，应该直接切换到已授权的数据源，而不是放宽完整性校验。

当任一免费 Provider 无法取得完整数据时，命令行以非零状态退出并输出 JSON 错误；API 则返回 `503`。这表示本次没有产生研究结论，不应把上一次结果误当作当天的前收盘筛选结果。

## 现在需要你提供的内容

框架和 `demo` 工作流已可运行。`eastmoney_free` 与 `sina_free` 提供无第三方 Python 依赖的免费快照与候选股日 K；其若无法获得完整快照，会拒绝生成推荐。接入完整生产链路前，需要确认：

1. 一个可用于 AKShare 原型网络访问的环境，或 Tushare Pro Token 与已购买的权限范围。
2. 若需要媒体新闻，提供具备 `major_news` 权限的 Tushare Token；未提供时仍会运行已接入的 CNINFO 公告，但不会虚构媒体催化。
3. 是否需要同一时刻历史量能。若需要，必须有分钟数据权限或授权行情源。
4. 选择一个 LLM API 并提供其安全配置方式；密钥只放在环境变量或密钥管理服务中，绝不写进 YAML 或仓库。

Level-2、逐笔成交、委托簿、竞价撤单均不属于当前 14:30 单次快照 MVP；未来若启用，需单独采购并核实授权。

## 明确排除的数据来源

- Kimi 内置或插件形式提供的同花顺数据不得被本项目自动读取、导出、缓存、转发或用于任何评分、回测和推荐。其可见性不等于项目获得了外部 API 或再利用授权。
- 同花顺及其他第三方页面数据只有在取得明确的、适用于本项目自动化用途的授权后，才能新增对应 Provider。
- AKShare 是数据接入库，不代表其每个上游页面都获得再利用授权；当前项目仅实现了东方财富行情/日 K 适配器。任何 AKShare 的 `*_ths` 或同花顺上游接口都保持禁用。
