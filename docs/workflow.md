# 前一交易日收盘筛选工作流

```text
# 收盘后：缓存完整全市场日 K 快照
  -> 下个交易日：读取最近的完整快照
  -> Market Scanner（ST、退市、价格、成交额、涨跌幅）
  -> 用户选定价格区间、创业板 / 科创板范围后通过基础规则的完整池（不按数量截断）
  -> Pattern Gate（对完整池计算最近完整日 K：阳线完美十字、阳线锤子）
  -> 展示全部形态命中结果（不加载历史数据）
  -> Research Pool（全部形态命中股）
  -> CNINFO 公告 + AKShare/东方财富个股新闻 + 可选 Tushare major_news（只对 Research Pool；来源链接/归因留档）
  -> K线趋势 / 量能 / 板块新闻 / 资金基本面风险
  -> LLM 研究（可选，结构化 JSON）
  -> Decision Agent（权重、阈值、否决）
  -> Markdown 报告 + SQLite 推荐记录
```

全市场扫描、形态计算和决策评分均为确定性 Python 代码。LLM 只能读取候选池的结构化证据，不允许全市场扫描或单独决定最终排序。

前端会把本次价格区间以及创业板、科创板开关传入后端。形态筛选、全市场审计视图和后续深度研究复用同一组已应用条件；创业板按 `300/301` 代码段识别，科创板按 `688/689` 代码段识别，主板与北交所不受这两个开关影响。

`WORKFLOW_NODES` 与 `WORKFLOW_EDGES` 位于 `orchestrator/definition.py`；它们是后续安装 LangGraph 后映射为图节点的稳定契约。当前同步工作流可直接运行，避免在尚未接入真实 API 时引入运行时依赖。

## 现有算法边界

- 已实现：基于最近完整交易日开、高、低、收盘价的阳线完美十字 / 阳线锤子数学识别。默认要求收盘高于开盘、日内振幅至少为开盘价的 3%；完美十字要求实体不超过总振幅 2%，上下影各占至少 45%，两者长度差不超过总振幅 6%；锤子线要求实体为 3%–30%、下影至少为实体两倍且占总振幅 60% 以上、上影不超过实体的一半。T 字线与不对称十字均被排除。全部阈值均可在 `configs/workflow.yaml` 调整；另有基础 MA 趋势、日量能比与可配置加权决策。
- 明确未完成：同一时刻历史量能曲线、ATR、支撑压力、板块共振、新闻语义情绪抽取、资金流归一化、个性化权重学习。

CNINFO 公告、AKShare/东方财富个股新闻和可选 Tushare `major_news` 已接入为候选级、可追溯的新闻证据；它们不是全市场扫描的一部分。东方财富结果还必须包含候选股代码或名称，避免宽泛搜索误归因；标题级的正负关键词只影响当前 `CatalystAgent` 的可解释默认分数，后续必须结合复盘统计再调权重。

未完成项必须接入相应数据并通过回测后才能进入推荐规则。
